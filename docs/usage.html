使い方

キーボードショートカット・トレイ操作・配列切り替え・設定ファイル・ログのリファレンス

入力モードの仕組み

awase は「IME の状態」と「エンジンの状態」の組み合わせで動作を切り替えます。

IME エンジン 動作
ON ON 親指シフト（NICOLA）入力
ON OFF 通常 IME 入力（ローマ字変換）
OFF どちらでも 英字直接入力（エンジンは自動停止）

IME が OFF のときはエンジンが自動的に停止するため、パスワード欄などでも誤動作しません。

キーボードショートカット

デフォルト設定でのショートカット一覧です。config.toml で変更できます。

カテゴリ キー 動作
IME Ctrl + 変換 IME を ON にする（冪等 — 既に ON なら何も起きない）
IME Ctrl + 無変換 IME を OFF にする（冪等 — 既に OFF なら何も起きない）
エンジン Ctrl + Shift + 変換 NICOLA エンジンを ON にする（親指シフト入力を有効化）
エンジン Ctrl + Shift + 無変換 NICOLA エンジンを OFF にする（ローマ字入力に切り替え）
エンジン Ctrl + Shift + F12 エンジンの ON/OFF をトグル（グローバルホットキー、config.toml で変更可）
システム Ctrl + Shift + F11 現在のウィンドウのフォーカス判定を強制上書き（テキスト入力 ↔ バイパスを切替）
親指シフト 変換 右親指シフトキー（エンジン ON 中、他のキーと同時押しで NICOLA 判定）
親指シフト 無変換 左親指シフトキー（エンジン ON 中、他のキーと同時押しで NICOLA 判定）
IME 半角/全角 IME の ON/OFF をトグル（Windows 標準の動作）

冪等な IME 制御について： Ctrl+変換 は「IME を ON にする」という操作です。従来のトグル方式と異なり、現在の状態がずれていても必ず意図した結果になります。 エンジン ON/OFF とは独立して動作します。

トレイアイコン操作

タスクバー右端のシステムトレイに awase アイコンが常駐します。

右クリックメニュー

  • 配列選択 — layouts_dir 内の .yab ファイルを一覧表示。選択するとレイアウトが即座に切り替わります
  • 有効/無効切替 — エンジンの ON/OFF をトグルします（Ctrl+Shift+F12 と同じ操作）
  • 管理者として再起動 — UAC で昇格して awase を再起動します。管理者権限を要求するアプリ（一部のゲームなど）でも動作するようになります
  • 設定... — 設定画面を開きます
  • 終了 — awase を終了します

アイコンの見方

  • アイコンが通常表示のとき — エンジン ON（親指シフト入力が有効）
  • アイコンがグレーアウトしているとき — エンジン OFF（通常入力）

設定画面（awase-settings）

トレイアイコン右クリック → 「設定...」で開きます。変更後は「適用」ボタンで保存・即時反映されます。

確定モードの選択肢

モード動作向いている人

IME 検出タブ

awase が IME 状態の変化を把握するためのキー設定です。通常はデフォルトのままで問題ありません。

項目説明
トグルキーOS が IME ON/OFF をトグルするキー（例: 半角/全角）
ON キーOS が IME を ON にするキー
OFF キーOS が IME を OFF にするキー

「デフォルトに戻す」ボタンで標準設定に戻せます。

config.toml の主要設定

awase の動作は config.toml で細かく調整できます。

ショートカットのカスタマイズ

# エンジン ON キー（NICOLA 入力を有効にするキー） engine_on_keys = ["Ctrl+Shift+VK_CONVERT"] # エンジン OFF キー（NICOLA 入力を無効にするキー） engine_off_keys = ["Ctrl+Shift+VK_NONCONVERT"] # IME ON キー（IME を ON にするキー） ime_on_keys = ["Ctrl+VK_CONVERT"] # IME OFF キー（IME を OFF にするキー） ime_off_keys = ["Ctrl+VK_NONCONVERT"] # エンジン ON/OFF トグルのグローバルホットキー engine_toggle_hotkey = "Ctrl+Shift+F12"

複数のキーを割り当てるには配列に追加してください: ["Ctrl+Shift+VK_CONVERT", "Ctrl+F8"]

同時打鍵の判定

[general] # 同時打鍵の判定閾値（ミリ秒）。NICOLA 規格標準は 100ms。 simultaneous_threshold_ms = 100 # 確定モードの選択 # ngram_predictive - n-gram 予測で投機/待機を動的切替（推奨） # wait - タイムアウトまで保留（最も正確） # speculative - 即時出力（最速） # two_phase - 短い待機→投機出力→差し替え # adaptive_timing - 打鍵速度に応じて閾値を自動調整 confirm_mode = "ngram_predictive"

親指キーのカスタマイズ

[general] # 左親指キー（VK コード名） left_thumb_key = "VK_NONCONVERT" # 無変換キー # 右親指キー（VK コード名） right_thumb_key = "VK_CONVERT" # 変換キー

キーボードモデルと出力方式

[general] # キーボードモデル（JIS キーボードを使っている場合はデフォルトで OK） # keyboard_model = "jis" # JIS キーボード（デフォルト） # keyboard_model = "us" # US キーボード # ローマ字出力の送信方式（通常は変更不要） # unicode - ひらがなを直接送信（IME 不要、推奨） # per_key - 1文字ずつ SendInput（他フックとの互換性重視） # batched - 全文字を1回の SendInput output_mode = "unicode"

新しい Microsoft IME（Windows 10 20H1 以降）では unicode モードを推奨します。per_key / batched では入力が正しく処理されない場合があります。

自動起動の設定

[general] # Windows ログオン時に自動起動する # "enabled" - 自動起動を有効にする # "disabled" - 自動起動を無効にする # "ask" - 初回起動時にダイアログで確認する（デフォルト） auto_start = "enabled"

Windows 11 をお使いの場合： スタートアップフォルダからの起動が失敗することがあります。その場合はタスクスケジューラで「ログオン時」トリガーのタスクを作成するか、レジストリの HKCU\Software\Microsoft\Windows\CurrentVersion\Run に登録してください。

特定ウィンドウの動作オーバーライド

[focus_overrides] # 特定アプリでテキスト入力を強制する force_text = [ { process = "myapp.exe", class = "MyWindowClass" }, ] # 特定アプリでエンジンを完全スキップする force_bypass = [ { process = "SearchHost.exe", class = "SearchHost" }, ]

スタートメニューや検索バーで誤動作する場合は force_bypass に追加してください。

アプリ別の出力モード上書き

[focus_overrides] # VK キーストロークで送信する（Chrome / Edge / VS Code 等 Electron ベースのアプリ） force_vk = [ { process = "chrome.exe", class = "Chrome_RenderWidgetHostHWND" }, { process = "code.exe", class = "Chrome_RenderWidgetHostHWND" }, ] # TSF ネイティブモードで送信する（WezTerm 等） force_tsf = [ { process = "wezterm-gui.exe", class = "org.wezfurlong.wezterm" }, ]

IME 検出キーの手動設定

通常は自動検出で十分ですが、特殊なキーボードや IME を使う場合に手動設定します。

[ime_sync] # IME ON/OFF をトグルするキー（例: 半角/全角） # toggle = ["VK_KANJI"] # IME を直接 ON にするキー # on = ["VK_DBE_HIRAGANA"] # IME を直接 OFF にするキー # off = ["VK_DBE_SBCSCHAR"]

緊急操作・リセット

入力がおかしくなったときや、エンジンを強制的に止めたいときの操作です。

パニックリセット — 状態を全初期化する

修飾キー（Shift・Ctrl など）が押しっぱなし状態でスタックした・IME 状態が混乱した・ 保留中の文字が残っているなど、入力全体がおかしくなったときに使います。

操作： Ctrl+無変換 → Ctrl+変換 → Ctrl+無変換 を 2秒以内 に素早く入力する

（IME-OFF → IME-ON → IME-OFF のシーケンス。同じキーの連打では発動しません）

効果：

  • エンジンの保留状態をフラッシュ（未確定の同時打鍵を破棄）
  • IME の未確定文字列をキャンセル
  • Shift・Ctrl・Alt・Win の押しっぱなし状態を強制解除
  • IME 状態を OFF → ON で再初期化
  • awase 内部の状態を全リセット

成功するとトレイバルーン通知「状態をリセットしました」が表示されます。

エンジン強制 OFF — 単キー3連打

Ctrl キーがスタックしているなど、通常のショートカットも効かない状態で NICOLA エンジンを強制的に止めたいときの最終手段です。 デフォルトでは無効です。config.toml で有効にしてください。

[general] # 単独で3回連打するとエンジン OFF になるキー（緊急脱出用） engine_off_solo_triple = "VK_NONCONVERT" # 無変換キーの場合

操作： 設定したキーを他のキーと同時押しせず、単独で 400ms 以内に3回 連打する

※ 通常の親指シフト入力中は「親指キー単独タイムアウト」としてカウントされるため、 タイピングの流れで誤発動しにくい設計になっています。

レイアウト（.yab）の切り替え

親指シフト配列は .yab ファイルで定義されています。複数の配列を使い分けることができます。

切り替え方法

  • トレイアイコン右クリック → 配列選択 — 最も手軽な方法。即座に切り替わります
  • 設定画面「基本設定」タブ — レイアウトのドロップダウンで選択 → 「適用」
  • config.toml を編集 — default_layout = "nicola.yab" でデフォルトを変更

カスタム配列を追加する

  1. 新しい .yab ファイルを config.toml の layouts_dir に指定したフォルダに配置します（デフォルトは config/）
  2. 設定画面「基本設定」タブの「再スキャン」ボタンを押します
  3. ドロップダウンに新しい配列が表示されるので選択して「適用」します

.yab 形式の詳細は GitHub の README を参照してください。

ログ・デバッグ

動作がおかしいときはログを確認してください。

ログファイルの場所

  • 通常起動時：awase.exe と同じフォルダの awase.log（起動のたびに追記されます）
  • Scoop でインストールした場合：%USERPROFILE%\scoop\apps\awase\current\awase.log

デバッグモードで起動する

詳細なログが必要な場合はコマンドプロンプトからデバッグモードで起動します。

awase.exe --debug

または環境変数 RUST_LOG=debug を設定してから起動しても同じです。

アンインストール

MSI インストーラー版

Windows の「設定 → アプリ」または「コントロールパネル → プログラムのアンインストール」から awase を選択してアンインストールします。

Scoop 版

scoop uninstall awase

ZIP 版（ポータブル）

  1. トレイアイコンを右クリック → 「終了」で awase を終了します
  2. 自動起動を設定している場合はタスクスケジューラから「awase」タスクを削除します
  3. 解凍したフォルダごと削除します

Microsoft IME のレジストリ設定を元に戻す（任意）

Microsoft IME を使用していた場合、awase が以下のレジストリ値を書き込んでいることがあります。 親指シフト以外の用途で 無変換/変換 の割り当てを戻したい場合は削除してください。

HKCU\Software\Microsoft\IME\15.0\IMEJP\MSIME IsKeyAssignmentEnabled KeyAssignmentMuhenkan KeyAssignmentHenkan

または Microsoft IME の「設定 → キーとタッチのカスタマイズ」で「各キーに好みの機能を割り当てる」をオフにしてください。

初期設定の手順

初めて使うときの推奨手順です。

1. 起動確認

  • awase を起動するとシステムトレイにアイコンが表示されます
  • テキストエディタなどを開き、メモ帳などに入力テストを行います
  • Ctrl+変換 で IME ON → 親指シフト入力が開始します

2. Microsoft IME を使う場合（初回のみ）

awase 起動時に Microsoft IME が検出されると、キー割り当ての自動設定ダイアログが1回だけ表示されます。 「OK」を押すと以下のレジストリ値を書き込み、無変換 で IME オフ・変換 で IME オンに割り当てます。

HKCU\Software\Microsoft\IME\15.0\IMEJP\MSIME IsKeyAssignmentEnabled = 1 ← キー割り当てを有効化 KeyAssignmentMuhenkan = 1 ← 無変換キー → IME オフ（直接入力） KeyAssignmentHenkan = 0 ← 変換キー → ひらがな（IME オン）
  • 既に同じ設定が書かれている場合はダイアログを表示せずスキップします
  • 「キャンセル」を押した場合はレジストリを変更しません
  • 設定を元に戻すには Microsoft IME の「設定 → キーとタッチのカスタマイズ」でキー割り当てを無効化してください

3. 確定モードの調整

  • まず confirm_mode = "ngram_predictive"（推奨）で試す
  • 誤変換が多い場合は "wait" に変更（正確だがやや遅め）
  • 高速タイピングで違和感がある場合は "adaptive_timing" を試す

4. Chrome / Edge での入力がうまくいかない場合

  • Google 日本語入力または Microsoft IME を使用していることを確認します（awase は両 IME に対応しています）
  • Ctrl+変換 で IME を ON にしてから入力を始めます
  • 最初の1文字だけ英字になる場合は awase が自動的に修正します（TSF cold-start 対応）

5. WezTerm で使う場合

awase は VK_IME_ON (0x16) / VK_IME_OFF (0x1A) を送って IME を制御します。 これらは実キーボードに存在しない仮想キーで VT エスケープシーケンスも生成しないため、 WezTerm や Windows Terminal での追加設定は不要です。

6. エンジンがおかしくなったときのリセット

  • Ctrl+Shift+変換 でエンジン ON を明示的に指定
  • Ctrl+変換 で IME ON を明示的に指定
  • それでも解決しない場合はトレイアイコンを右クリック → 「有効/無効切替」でトグルする
