# ADR-003: フックからブロッキング IME 検出を追い出し、キャッシュ化

## ステータス
採用

## コンテキスト
`WH_KEYBOARD_LL` フックコールバック内で `SendMessageTimeoutW`（クロスプロセス IPC）を毎キーイベント呼んでいた。IME 遷移直後にこの呼び出しが最大 200ms ブロックし、後続キーの `KBDLLHOOKSTRUCT.time` タイムスタンプが歪む。NICOLA の同時打鍵判定閾値 (100ms) を超え、「かわ」が「かほ」になる等の誤判定が発生。

タイムアウトを 10ms に縮めたが、対症療法に過ぎなかった。

## 決定
フックコールバックから IME 検出の呼び出しを完全に除去し、キャッシュ方式に移行:

- `IME_STATE_CACHE` (`AtomicU8`): メッセージループで更新、フックで読むだけ (1ns)
- 更新トリガー:
  - フォーカス変更（デバウンス 50ms）
  - `WM_INPUTLANGCHANGE`
  - IME トグルキー検出 → `PostMessageW` → 即時更新
  - 500ms ポーリング（安全ネット）
- `SendMessageTimeoutW` のタイムアウトは 50ms に復帰（メッセージループ上なのでブロッキング OK）

## 結果
- フックのブロッキング完全解消 → タイムスタンプ歪み消滅
- フック内の IME 検出コード 120行 → 15行に削減
- IME 遷移直後の同時打鍵誤判定が構造的に解消

## 却下した代替案
- タイムアウト短縮 (10ms): 対症療法。ポーリング時にタイムアウトしやすくキャッシュが不正確に
- フック内で非同期検出: LL フックは同期コールバックのため不可能
