# ADR-018: 他の親指シフトエミュレータからの教訓と対策

## ステータス

採用

## コンテキスト

やまぶきR、紅皿（Benizara）、DvorakJ、親指の友 Mk-II 等の既存実装で報告された問題を調査し、awase での対応状況を整理した。

## 調査結果と対応

### 対応済み（コード変更）

| 問題 | 他ソフトの状況 | awase の対応 |
|------|---------------|-------------|
| **フック消失** | 紅皿で「PC高負荷時にキーボードフリーズ」。Windows は300ms超過11回でフック無言削除 | ハートビート監視 + 10秒無応答で警告通知。処理時間100ms超で警告ログ |
| **管理者権限プロセス (UIPI)** | 紅皿 v0.1.3 で管理者昇格オプション追加 | トレイメニューに「管理者として再起動」追加。非管理者時は起動時に警告 |
| **キーボードレイアウト変更** | 2025年4月 Windows Update で106→101に変更される事例多発 | 起動時に日本語レイアウトチェック。英語キーボード検出時は設定方法を案内。WM_INPUTLANGCHANGE でも監視 |
| **画面ロック/セッション切替** | やまぶきRでスリープ後に動作しなくなる報告 | WM_POWERBROADCAST + WM_WTSSESSION_CHANGE で全状態リフレッシュ |
| **修飾キースタック** | 全エミュレータ共通の問題 | フォーカス変更時に GetAsyncKeyState で修飾キー状態を同期 |
| **カタカナ/ひらがなキー** | lock型で KeyUp が来ない。親指キーに使えない | config バリデーションで VK_KANA 設定時に警告。IMEガードの10キー上限で保護 |

### 対応済み（ドキュメント/設定）

| 問題 | 対応 |
|------|------|
| **新 Microsoft IME 互換性** | Unicode モード（デフォルト）で IME 非依存。config.sample.toml に推奨と旧 IME 設定の案内を記載 |
| **Windows 11 自動起動失敗** | タスクスケジューラ/レジストリ Run キーの代替方法を config.sample.toml に記載 |
| **スタートメニュー/検索での不具合** | force_bypass の設定例を config.sample.toml に記載（SearchHost, StartMenuExperienceHost 等） |

### 既存設計で対処済み

| 問題 | 対処方法 |
|------|---------|
| **IME 状態のポーリング** | 500ms タイマーで定期的にクロスプロセス検出（マウスでの IME 操作もカバー） |
| **自己注入ループ防止** | INJECTED_MARKER (0x4B45_594D) で SendInput の自己ループを防止 |
| **アプリ別無効化** | config の force_bypass + ウィンドウごとのエンジン状態記憶 |
| **RDP タイミング** | simultaneous_threshold_ms が設定可能（ユーザーが調整） |

### 対応不要/対応困難

| 問題 | 理由 |
|------|------|
| **SendInput の SendEvent 劣化** | 他のキーボードフックが存在すると SendInput がアトミック性を失う。検出・対策とも困難。影響は限定的（出力の順序がずれる可能性）|
| **タッチキーボード** | WH_KEYBOARD_LL で通常のキーイベントとして捕捉される。特別な対応不要 |
| **カーネルドライバ方式** | 親指の友 Mk-II のようなカーネルドライバは UIPI 制限がないが、署名やメンテナンスコストが高い。ユーザーモードフック + 管理者昇格で十分 |

## 設計原則（他ソフトの教訓から）

1. **フックコールバックは100ms以内に返す** — Windows は300ms超過11回でフックを無言削除する
2. **IME 状態はポーリングで補完する** — イベント駆動だけでは取りこぼす（Google 日本語入力が不正確な値を返す等）
3. **Unicode モードを推奨する** — IME の内部変更（新旧 IME 切替等）に影響されない
4. **管理者権限は必須ではなくオプション** — 通常使用では不要、必要な場合のみ昇格
5. **Windows Update による破壊に備える** — キーボードレイアウト設定が変更される可能性を検出・警告
